###Final Project
	使用https://github.com/caochun/huluwa的代码框架及部分接口
	使用maven作为自动构建工具,打包生成了可执行jar文件

###结构说明
	Species: 从Thing2D处继承得来,
		每一个物种具有战斗能力(Fight),生存属性(Dead),血量上限(Blood),
		在每一次战斗之后血量将会有相应减少(根据战斗种类相异)
		
					|---Scorpion(蝎子精)
					|----Grandpa(爷爷)
		 其子类-----|----Huluwa(葫芦兄弟)
					|----Snake(蛇精)
					|----Lackey(小喽)

	Tile:用来作背景覆盖

	Ground:初始化Jframe,并在原有的基础上加了一个输出帮助,调试信息的文本框

	Audience:远离战场的记录员,继承了Spieces,执行记录战斗情况以及加载战斗的功能,记录文件在工程目录下record.txt文件

	Referee:远离战场的裁判员,继承了Species,执行判定战斗胜负的功能

	Fight:初始化Jpanel,对键盘做出响应,并描述战斗机制--------|各种族如何找寻敌人
												          |找到敌人之后战斗胜负如何判定

###战斗机制
	各种族将会试探性地寻找战场上离自己最近的敌人,朝其接近
	
	相遇后将会发生战斗,每个物种血量越高,胜算越大,
	由于葫芦娃只有7个,敌对阵营人多势众,考虑到战斗的平衡
	葫芦娃将具有强大的战力
	但是也存在相生相克,
	蛇精对葫芦娃来说相对胜算较大,蝎子精次之,小喽即便满血,对上葫芦娃也是以卵击石
	而爷爷虽然德高望重,可岁月不饶人,故爷爷对上所有敌人均胜算较低
	

###设计原则
	每个类均有着自己的功能,
	类可以被继承,方法可以被重写,但是不能被随意修改
	遵循了"SRP","OCP"原则

	同时遵守了里氏替换原则Spieces,Thing2D等类均为动态结构,
	保持了设计的封闭性,其衍生类均可以替换基类
	也较好地体现了继承,封装,多态等特性
	
###线程协同
	当各战斗生物可能要独占某种资源(比如到达战场某位置,与其他生物战斗等)时,
	使用了lock来避免可能发生的冲突
